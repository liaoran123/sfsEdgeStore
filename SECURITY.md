# 安全策略

## 报告安全漏洞

我们非常重视安全问题。如果您发现了安全漏洞，请通过以下方式报告：

- **GitHub Issues**：请在 [GitHub Issues](https://github.com/your-username/sfsEdgeStore/issues) 中报告，并标记为 `security`
- **电子邮件**：如有敏感信息，请通过 [您的邮箱] 联系我们

## 漏洞响应流程

1. **确认接收**：我们会在 48 小时内确认收到您的报告
2. **分析与修复**：我们会评估漏洞的严重性并开发修复方案
3. **发布更新**：修复完成后，我们会发布安全更新
4. **感谢**：我们会在安全公告中感谢您的贡献（如果您同意）

## 支持的版本

我们目前支持以下版本的安全更新：

| 版本 | 支持状态 |
|------|---------|
| 最新版本 | ✅ 完全支持 |
| 前一个主要版本 | ⚠️ 关键安全修复 |
| 更早版本 | ❌ 不再支持 |

## 安全最佳实践

### 部署安全

- 使用 TLS/HTTPS 加密通信
- 配置强密码和 API Key
- 定期更新到最新版本
- 限制网络访问来源

### 运行时安全

- 监控系统日志
- 定期备份数据
- 遵循最小权限原则
