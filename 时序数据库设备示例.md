# sfsDb时序数据库设备数据管理与查询示例

## 简介

时序数据库是一种专门用于存储和查询时间序列数据的数据库系统，广泛应用于物联网、监控系统、金融交易等领域。在这些场景中，数据通常与时间密切相关，并且需要高效的范围查询能力。本文将介绍如何使用 sfsDb 数据库系统构建一个简单的时序数据库，用于管理设备传感器数据。

## 系统设计

### 数据模型设计

对于设备数据，我们采用以下数据模型：

- **设备ID (device_id)**: 标识不同的设备
- **时间戳 (timestamp)**: 记录数据采集的时间
- **温度 (temperature)**: 设备采集的温度值
- **湿度 (humidity)**: 设备采集的湿度值
- **压力 (pressure)**: 设备采集的压力值

### 索引设计

为了支持高效的设备数据查询，我们采用组合主键设计：

- **组合主键**: `device_id` + `timestamp`
- **设计原理**: 
  - 第一个字段 `device_id` 用于精确匹配特定设备
  - 第二个字段 `timestamp` 用于范围查询特定时间段的数据
  - 这种设计可以高效地支持 "查询某个设备在某个时间段内的数据" 这类常见查询

```go
err = table.SetFields(fields)

```

```go

    t.Fatalf("创建索引失败: %v", err)
```

### 5. 执行范围查询

```go
// 测试组合主键范围搜索 - 查询device_001设备的时间段数据
searchImpl := NewSearchImpl(table)
defer GlobalSearchImplPool.Put(searchImpl)

// 测试范围搜索: device_id="device_001", timestamp从1609462800到1609470000
// 注意：组合主键时，第一个字段必须全量匹配，最后一个字段是范围
iter, err := searchImpl.SearchRange(nil, &map[string]any{"device_id": "device_001", "timestamp": 1609462800}, &map[string]any{"device_id": "device_001", "timestamp": 1609470000})
if err != nil {
    t.Fatalf("范围搜索失败: %v", err)
}
defer iter.Release()
```

### 6. 处理查询结果

```go
// 获取记录
records := iter.GetRecords(true)
defer records.Release()

// 验证结果
expectedCount := 3 // 应该返回device_001的3条记录
if len(records) != expectedCount {
    t.Errorf("期望返回 %d 条记录，实际返回 %d 条", expectedCount, len(records))
}

// 验证返回的记录包含预期的device_id和timestamp值
for _, record := range records {
    // 验证device_id
    deviceID, ok := record["device_id"].(string)
    if !ok {
        t.Errorf("记录中的device_id字段类型不是string: %T", record["device_id"])
        continue
    }
    if deviceID != "device_001" {
        t.Errorf("记录中的device_id不是device_001: %s", deviceID)
    }

    // 验证timestamp在范围内
    timestamp, ok := record["timestamp"].(int)
    if !ok {
        t.Errorf("记录中的timestamp字段类型不是int: %T", record["timestamp"])
        continue
    }
    if timestamp < 1609462800 || timestamp > 1609470000 {
        t.Errorf("记录中的timestamp不在范围内: %d", timestamp)
    }
}
```

## 技术要点解析

### 组合主键的优势

1. **高效查询**: 组合主键将设备ID和时间戳结合在一起，使得查询特定设备的时间段数据非常高效
2. **数据组织**: 相同设备的数据在物理存储上是连续的，有利于顺序读取
3. **范围查询**: 时间戳作为组合主键的一部分，支持高效的范围查询

1. **组合主键规则**: 当使用组合主键进行范围查询时，第一个字段必须全量匹配，最后一个字段是范围
2. **时间戳选择**: 建议使用 Unix 时间戳（秒或毫秒）作为时间字段，便于范围查询
3. **数据分区**: 对于大规模设备数据，可以考虑按时间或设备进行数据分区，进一步提高查询性能
4. **索引优化**: 根据查询模式调整索引设计，确保最常见的查询能够使用索引

## 最佳实践

### 1. 数据采集策略

- **采样频率**: 根据业务需求合理设置数据采集频率，避免数据过载
- **数据压缩**: 对于高频采集的数据，考虑使用数据压缩技术减少存储空间
- **数据预处理**: 在数据插入前进行必要的预处理，如异常值检测、数据清洗等

### 2. 查询优化

- **时间范围限制**: 查询时尽量指定合理的时间范围，避免全表扫描
- **设备ID精确匹配**: 总是在查询中指定设备ID，充分利用组合主键的优势
- **批量查询**: 对于多个设备的查询，考虑使用批量查询接口提高效率

### 3. 存储管理

- **数据 retention**: 设置合理的数据保留策略，定期清理过期数据
- **备份策略**: 实施定期备份策略，确保数据安全
- **监控告警**: 监控数据库性能和存储空间，设置合理的告警阈值
