## 概述

sfsEdgeStore 和 sfsDb 的加密功能是**可选的、灵活的**，您可以根据实际场景需求选择是否启用加密。

### 核心原则

- ✅ **不是所有场景都必须加密**
- ✅ **根据数据敏感度选择**
- ✅ **在安全和性能之间取得平衡**
- ✅ **加密是可选功能，可以随时切换**

---

## 场景加密需求评估

| 场景 | 是否需要加密 | 推荐级别 | 说明 |
|------|-------------|---------|------|
| **边缘计算节点** | ⚠️ 可选但推荐 | ⭐⭐⭐ | 取决于数据敏感度 |
| **IoT 网关设备** | ⚠️ 可选但推荐 | ⭐⭐⭐⭐ | 取决于设备部署环境 |
| **智能终端设备** | ✅ 强烈推荐 | ⭐⭐⭐⭐⭐ | 设备可能丢失或被盗 |

---

## 详细场景分析

### 1. 边缘计算节点

**部署环境**：
- 通常在相对安全的机房或机柜中
- 物理访问受限
- 网络环境相对可控

**加密需求**：⭐⭐⭐

**建议**：

| 数据类型 | 加密建议 |
|---------|---------|
| **敏感工业数据** | ✅ **启用加密** |
| **普通监控数据** | ⚡ **可选择不加密以提升性能** |
| **混合场景** | 🔄 **根据数据分级选择** |

**适用场景**：
- 工厂核心数据采集节点
- 数据中心边缘节点
- 机房内的聚合节点

---

### 2. IoT 网关设备

**部署环境**：
- 可能在工厂车间、户外机柜等
- 物理访问可能受限或公开
- 环境条件可能复杂

**加密需求**：⭐⭐⭐⭐

**建议**：

| 部署环境 | 加密建议 |
|---------|---------|
| **物理安全可控的环境** | ⚠️ **可选** |
| **易接触或公共区域** | ✅ **建议启用** |
| **户外部署** | ✅ **强烈建议启用** |

**适用场景**：
- 工厂车间网关
- 户外智能柜
- 园区聚合网关
- 楼宇自动化网关

---

### 3. 智能终端设备 ⚠️ 高风险

**部署环境**：
- 手持终端
- 移动设备
- 可能丢失或被盗
- 用户环境复杂

**加密需求**：⭐⭐⭐⭐⭐

**建议**：**必须启用加密**

**理由**：
- 设备可能被盗
- 本地数据包含敏感信息
- 符合数据保护法规要求（GDPR、网络安全法等）
- 保护用户隐私和企业数据

**适用场景**：
- 手持巡检终端
- 移动数据采集器
- 便携式检测设备
- 可穿戴设备

## 配置建议

### 配置矩阵

| 场景 | 性能优先 | 安全优先 | 平衡方案 |
|------|---------|---------|---------|
| **边缘计算节点**<br>(机房内) | `Enabled: false` | `Enabled: true` | `Enabled: true`<br>`Algorithm: AES-256-GCM` |
| **IoT 网关**<br>(车间/户外) | `Enabled: false` | `Enabled: true` | `Enabled: true`<br>`MasterKey: 预共享密钥` |
| **智能终端**<br>(高风险) | ❌ 不推荐 | `Enabled: true`<br>`Password: 用户密码` | `Enabled: true`<br>`Password + Salt` |

---

### 详细配置建议

#### 场景 A：资源受限 + 非敏感数据

**目标**：最大化性能，不处理敏感数据

**配置**：
```json
{
  "DBUseEncryption": false
}
```

**适用场景**：
- 公共环境监测数据
- 非敏感的设备状态监控
- 测试和开发环境
- 性能基准测试

**优点**：
- ⚡ 最高性能
- 🚀 最快启动速度
- 💾 最小存储开销
- 🔧 最简单配置

---

#### 场景 B：一般场景

**目标**：平衡安全和性能

**配置**：
```json
{
  "DBUseEncryption": true,
  "DBEncryptionKey": "your-32-byte-master-key-here",
  "DBEncryptionAlgorithm": "AES-256-GCM"
}
```

**适用场景**：
- 企业内部边缘节点
- 安全可控的 IoT 网关
- 一般工业数据采集
- 楼宇自动化系统

**优点**：
- ✅ 良好的安全性
- ⚡ 可接受的性能
- 🔧 配置相对简单
- 📦 易于部署

---

#### 场景 C：高安全要求

**目标**：最高安全性，处理敏感数据

**配置**（使用密码派生）：
```json
{
  "DBUseEncryption": true,
  "DBEncryptionKey": "derived-from-password",
  "DBEncryptionAlgorithm": "AES-256-GCM"
}
```

**适用场景**：
- 智能终端设备（高风险）
- 处理个人身份信息（PII）
- 金融相关数据
- 医疗健康数据
- 需要符合合规要求的场景

**优点**：
- 🔒 最高安全性
- 🔐 密码保护，密钥不硬编码
- 📜 符合数据保护法规
- 🛡️ 设备被盗时数据安全

## 性能与安全权衡

### 加密对性能的影响

| 指标 | 不加密 | 加密（AES-256-GCM） | 性能影响 |
|------|--------|-------------------|---------|
| **写入延迟** | 基准 | +5-15% | 轻微增加 |
| **读取延迟** | 基准 | +3-10% | 轻微增加 |
| **CPU 使用率** | 基准 | +2-5% | 可接受 |
| **存储开销** | 基准 | +5-10% | 可接受 |
| **启动时间** | 基准 | +10-20% | 可接受 |

### 何时选择不加密

选择不加密的情况：
- ✅ 数据完全不敏感
- ✅ 性能要求极高
- ✅ 资源极度受限（< 100MHz CPU）
- ✅ 测试和开发环境
- ✅ 短期数据，定期清理

### 何时选择加密

选择加密的情况：
- ✅ 处理敏感数据
- ✅ 设备可能丢失或被盗
- ✅ 需要符合数据保护法规
- ✅ 客户要求加密
- ✅ 企业安全政策要求

## 密钥管理

### 密钥安全最佳实践

#### 1. 密钥生成

```bash
# 生成 32 字节（256 位）的随机密钥
openssl rand -hex 32
```

#### 2. 密钥存储

| 环境 | 存储方式 | 安全等级 |
|------|---------|---------|
| **开发环境** | 环境变量或配置文件（.gitignore） | ⭐⭐ |
| **测试环境** | 密钥管理服务（Vault） | ⭐⭐⭐ |
| **生产环境** | 硬件安全模块（HSM）或云 KMS | ⭐⭐⭐⭐⭐ |

#### 3. 密钥轮换

**建议**：
- 定期轮换密钥（每 6-12 个月）
- 使用 sfsEdgeStore 的 `RotateEncryptionKey` API
- 保留旧密钥的备份，直到确认所有数据都已重新加密

#### 4. 密钥备份

- ✅ 离线备份密钥
- ✅ 使用安全的备份介质
- ✅ 分散存储备份
- ✅ 定期测试备份恢复

## 常见问题

### Q1: 加密会影响多少性能？

**A**: 通常影响在 5-15% 之间。对于大多数边缘场景，这个影响是可以接受的。如果性能要求极高，可以考虑不加密。

### Q2: 可以在加密和不加密之间切换吗？

**A**: 可以！但需要注意：
- 从非加密切换到加密：需要导出数据，重新创建加密数据库，再导入数据
- 从加密切换到非加密：同样需要数据导出导入
- 建议在部署前确定好方案

### Q3: AES-256-GCM 是什么？

**A**:
- AES-256：高级加密标准，256 位密钥
- GCM：Galois/Counter Mode，提供认证加密
- 同时提供保密性和完整性保护
- 业界标准，广泛使用

### Q4: 如果忘记加密密钥怎么办？

**A**: ⚠️ **数据将无法恢复！**
- 请务必妥善保管密钥
- 做好密钥备份
- 考虑使用密钥管理系统

### Q5: 加密后数据库文件变大了？

**A**: 正常现象！加密会有一定的存储开销（通常 5-10%），这是为了存储加密元数据和认证标签。

### Q6: 可以只加密部分数据吗？

**A**: 当前版本是全库加密。如果需要部分加密，可以考虑：
- 在应用层对敏感字段加密
- 使用分级存储，敏感数据放加密库
- 非敏感数据放普通库

## 总结

### 核心要点

1. **加密是可选的**：根据实际需求选择
2. **性能 vs 安全**：找到适合场景的平衡点
3. **场景驱动**：不是所有场景都需要加密
4. **密钥管理**：加密的安全性取决于密钥管理
5. **灵活配置**：可以随时调整策略

### 决策流程图

```
开始
  ↓
数据是否敏感？
  ├─ 否 → 不加密（性能优先）
  └─ 是 → 设备是否安全？
            ├─ 否 → 必须加密（高安全）
            └─ 是 → 数据是否受规管？
                      ├─ 是 → 必须加密（合规）
                      └─ 否 → 可选加密（平衡）
```
