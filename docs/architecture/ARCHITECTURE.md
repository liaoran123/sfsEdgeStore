## 系统概述

### 设计目标

sfsEdgeStore 是一个轻量级边缘计算数据存储适配器，专为资源受限的边缘设备设计。其核心设计目标包括：

- **轻量级**: 内存占用 < 50MB，CPU 使用率 < 5%
- **高可靠性**: 断电恢复、数据重试、本地优先
- **易于部署**: 单二进制文件，无外部依赖
- **企业级**: 内置认证、监控、告警、备份功能

### 核心特性

- **MQTT 数据接入**: 订阅 EdgeX Foundry 事件主题
- **本地数据存储**: 使用 sfsDb (LevelDB 封装) 高效存储
- **实时监控**: 内置系统指标和业务指标监控
- **智能告警**: 阈值告警和异常检测
- **数据队列**: 断电恢复和数据重试机制
- **数据分析**: 内置时间窗口聚合和预测
- **认证授权**: API Key 和 RBAC 权限控制
- **HTTP API**: RESTful 接口供外部查询
- **数据同步**: 可选的数据上云同步
- **数据保留**: 自动清理过期数据

## 核心模块

### 1. MQTT 客户端 (mqtt/)

**职责**:
- 连接 MQTT Broker
- 订阅 EdgeX 事件主题
- 接收并解析 EdgeX 消息
- 将消息推送到数据队列

**核心文件**:
- `mqtt/client.go` - MQTT 客户端实现
- `mqtt/mapPool.go` - 对象池优化

**关键特性**:
- 自动重连机制
- QoS 1 消息保证
- TLS 支持
- 连接状态监控

### 2. 数据队列 (queue/)

**职责**:
- 缓存待处理数据
- 提供持久化队列
- 处理失败重试
- 断电恢复支持

**核心文件**:
- `queue/queue.go` - 队列实现
- `queue/queue_test.go` - 队列测试

**关键特性**:
- 持久化存储（防止数据丢失）
- 批量处理
- 指数退避重试
- 优雅关闭

### 3. 数据库 (database/)

**职责**:
- sfsDb (LevelDB) 封装
- 数据 CRUD 操作
- 批量插入
- 数据加密
- 事务支持

**核心文件**:
- `database/database.go` - 数据库操作
- `database/database_test.go` - 数据库测试
- `database/encryption_test.go` - 加密测试

**关键特性**:
- AES-256-GCM 加密
- 批量插入优化
- 多种数据库场景配置（embedded/iot/edge/game）
- 数据压缩

**数据库场景配置**:

| 场景 | 写缓冲 | 块缓存 | 适用场景 |
|------|--------|----------|----------|
| embedded | 2MB | 4MB | 嵌入式设备 |
| iot | 4MB | 8MB | IoT 设备 |
| **edge** (默认) | 16MB | 32MB | 边缘计算 |
| game | 64MB | 128MB | 游戏场景 |

### 4. HTTP 服务器 (server/)

**职责**:
- 提供 RESTful API
- 数据查询接口
- 健康检查端点
- 指标暴露
- 速率限制

**核心文件**:
- `server/server.go` - HTTP 服务器
- `server/ratelimit.go` - 速率限制

**API 端点**:
- `GET /health` - 健康检查
- `GET /metrics` - 系统指标
- `GET /alerts` - 告警信息
- `GET /api/readings` - 数据查询
- `POST /api/backup` - 创建备份
- `POST /api/restore` - 恢复备份
- `POST /api/auth/*` - 认证相关

### 5. 监控 (monitor/)

**职责**:
- 系统指标收集
- 业务指标统计
- Prometheus 格式输出
- 告警触发

**核心文件**:
- `monitor/monitor.go` - 监控实现

**监控指标**:
- **系统指标**: CPU、内存、磁盘、Goroutine 数
- **业务指标**: 消息接收数、处理数、队列长度、总读数
- **数据库指标**: 大小、读取次数、写入次数

### 6. 告警 (alert/)

**职责**:
- 阈值检查
- 告警通知
- 多渠道支持（Webhook、Email、Slack、PagerDuty）

**核心文件**:
- `alert/notifier.go` - 告警通知器

**告警级别**:
- **Critical**: 系统不可用
- **Warning**: 性能降级
- **Info**: 信息通知

### 7. 认证授权 (auth/)

**职责**:
- API Key 管理
- RBAC 权限控制
- 认证中间件
- API Key 清理

**核心文件**:
- `auth/api_key.go` - API Key 管理
- `auth/auth.go` - 认证核心
- `auth/middleware.go` - 认证中间件
- `auth/rbac.go` - 角色权限

**角色权限**:

| 角色 | 权限 |
|------|------|
| admin | 所有权限 |
| user | 读写数据、备份 |
| readonly | 只读数据 |

### 8. 数据保留 (retention/)

**职责**:
- 定期清理过期数据
- 批量删除优化
- 可配置保留策略

**核心文件**:
- `retention/retention.go` - 保留策略管理器

**配置项**:
- `RetentionDays`: 数据保留天数
- `CleanupInterval`: 清理间隔（小时）
- `CleanupBatchSize`: 批量删除大小

### 9. 数据同步 (sync/)

**职责**:
- 可选的数据上云同步
- 批量同步优化
- 失败重试
- 增量同步

**核心文件**:
- `sync/sync.go` - 同步管理器

### 10. 资源监控 (resource/)

**职责**:
- CPU 使用率监控
- 内存使用率监控
- 磁盘使用率监控
- 资源限制告警

**核心文件**:
- `resource/monitor.go` - 资源监控器

### 11. 数据分析 (analyzer/)

**职责**:
- 时间窗口聚合
- 数据预测
- 异常检测

**核心文件**:
- `analyzer/analyzer.go` - 分析引擎
- `analyzer/types.go` - 类型定义

### 12. 时间序列处理 (time/)

**职责**:
- 时间窗口管理
- 数据聚合
- 数据压缩
- 粒度管理

**核心文件**:
- `time/window.go` - 时间窗口
- `time/aggregationResult.go` - 聚合结果
- `time/compression.go` - 数据压缩
- `time/granularity.go` - 粒度管理
- `time/time_query.go` - 时间查询

### 13. 备份恢复 (backup/)

**职责**:
- 数据库备份
- 数据恢复
- 备份验证

**核心文件**:
- `backup/backup.go` - 备份管理器

### 14. 配置管理 (config/)

**职责**:
- 配置加载
- 配置验证
- 配置热更新
- 环境变量支持

**核心文件**:
- `config/config.go` - 配置核心
- `config/file.go` - 文件配置
- `config/env.go` - 环境变量
- `config/edgex.go` - EdgeX 集成

### 15. 管理 Agent (agent/)

**职责**:
- 极简管理接口
- 远程管理支持
- 状态查询

**核心文件**:
- `agent/agent.go` - Agent 实现
- `agent/readme.md` - Agent 文档

### 16. 模拟器 (simulator/)

**职责**:
- 模拟 EdgeX 数据
- 测试和开发使用
- 可配置模拟参数

**核心文件**:
- `simulator/simulator.go` - 模拟器实现

## 数据流

### EdgeX 数据接收流程

```
1. EdgeX Foundry 发布事件到 MQTT Broker
   │
   ▼
2. MQTT Client 接收消息
   │
   ▼
3. 消息解析和验证
   │
   ▼
4. 消息推送到数据队列
   │
   ├─────────────────┐
   │                 │
   ▼                 ▼
5a. 队列处理协程    5b. 监控更新
   │                 │
   ▼                 ▼
6a. 批量插入数据库  6b. 指标统计
   │
   ▼
7a. 成功确认 / 失败重试
```

```
1. 客户端发送 HTTP 请求
   │
   ▼
2. 认证中间件验证
   │
   ▼
3. 权限检查 (RBAC)
   │
   ▼
4. 参数验证和解析
   │
   ▼
5. 数据库查询
   │
   ▼
6. 结果格式化 (JSON/CSV)
   │
   ▼
7. 返回响应
```

## 技术选型

### 核心技术栈

| 技术 | 选型 | 说明 |
|------|------|------|
| **编程语言** | Go 1.25+ | 高性能、并发友好、静态链接 |
| **数据库** | sfsDb (LevelDB) | 轻量级、嵌入式、高性能 |
| **消息协议** | MQTT 3.1.1/5.0 | 物联网标准、轻量级 |
| **Web 框架** | 标准库 net/http | 无依赖、高性能 |
| **序列化** | JSON | 标准、易调试 |
| **日志** | 标准库 log | 简单、可扩展 |

### 依赖库

| 库 | 用途 | 来源 |
|----|------|------|
| `github.com/syndtr/goleveldb/leveldb` | LevelDB 封装 | sfsDb 基础 |
| `github.com/eclipse/paho.mqtt.golang` | MQTT 客户端 | MQTT 通信 |

## 扩展性设计

### 水平扩展

sfsEdgeStore 支持通过以下方式进行水平扩展：

1. **数据分区**: 按设备或时间范围分区
2. **消息分片**: 使用 MQTT 主题分片
3. **负载均衡**: 多个实例共享负载

### 垂直扩展

通过配置优化进行垂直扩展：

1. **数据库缓存**: 增加 `DBBlockCacheSize`
2. **队列大小**: 增加 `QueueSize`
3. **并发处理**: 增加处理协程数

### 插件机制

（预留设计，未来版本）

- 自定义数据处理器
- 自定义告警通知器
- 自定义数据同步器

## 设计原则

### 1. 轻量级优先

- 最小化内存占用
- 最小化 CPU 使用
- 最小化依赖

### 2. 可靠性优先

- 本地数据不丢失
- 断电自动恢复
- 失败自动重试

### 3. 简单性优先

- 单二进制部署
- 配置简单直观
- API 设计简洁

### 4. 可观测性优先

- 内置监控指标
- 详细日志输出
- 健康检查端点
